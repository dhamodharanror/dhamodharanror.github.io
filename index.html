How I Think About Systems

Architecture that serves the product

Real patterns from a production multi-tenant SaaS — not textbook diagrams.

Grape API (v1/v2) + Rails Controllers
↓ bearer token + CanCanCan RBAC
Service Objects · Lib Modules (117+)
↓ query abstraction
Repository Pattern · Model Concerns
↓ apartment middleware per request
PostgreSQL — schema-per-tenant
↓ after_commit hooks
Elasticsearch + ClickHouse
↓ async via Delayed Job
75+ Background Workers (cron + on-demand)

Schema-Based Multi-Tenancy

Every tenant gets an isolated PostgreSQL schema. Apartment middleware switches context per request via subdomain. Jobs carry tenant context automatically — no manual Tenant.switch().

Dual Analytics Pipeline

Elasticsearch for real-time search and segment filtering. ClickHouse ingests every 5 minutes for OLAP aggregations — survey trends, benchmarking, cross-tenant analytics.

Feature Flag Architecture

100+ AccountSetting flags gate every feature per tenant. Zero hardcoded conditionals. Features toggle per tenant without deploys — the only right way to do this at SaaS scale.

Integration Ecosystem

15+ HRIS and channel integrations behind a provider factory pattern. BambooHR, Workday, Keka, Darwinbox, Slack, Teams, WhatsApp. Adding a new integration doesn't touch existing ones.
