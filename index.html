01
Architecture

Multi-tenant systems that actually scale.

  • Schema-per-tenant PostgreSQL — full isolation per customer
  • Tenant context switches per request automatically
  • Background jobs carry tenant context
  • Zero cross-tenant exposure at scale
  • Participant flows, autosave, submission — clean within the model
02
Craft

Clean code, clean systems.

  • Repository pattern — no logic leaking into models
  • Service objects strictly for third-party wrappers
  • Concerns for shared model/controller logic
  • I love improving it once things are working

"Tech should serve the product, not the other way around."
